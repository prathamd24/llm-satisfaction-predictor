# 🤖 LLM Satisfaction Predictor

LLM Satisfaction Predictor is an end-to-end Machine Learning application that predicts the expected user satisfaction score of an LLM interaction based on its configuration and usage characteristics.

The system uses an XGBoost regression model trained on historical LLM usage data. Users provide details such as the selected LLM model, application domain, task type, prompt length, temperature, top-p, and whether RAG is enabled. The trained model then predicts the expected satisfaction score on the dataset's 3–5 scale.

The project is built as a complete production-style ML application with a FastAPI backend, a sleek modern web frontend, Firebase Google Authentication, automated testing, Docker support, and cloud deployment.

## 🔍 Problem Statement

Different LLM configurations can result in different levels of user satisfaction. This project uses historical LLM usage data to learn patterns between configuration parameters and user satisfaction.

The model uses features such as:
- LLM model
- Application domain
- Task type
- Prompt length
- Token configuration
- Temperature
- Top-p
- RAG enabled/disabled

and predicts: **Expected User Satisfaction Score** on a 3–5 scale.

## 💻 Local Setup

### Prerequisites
- Python 3.12+
- Dataset file in `data/raw/genai_llm_usage_dataset_1000.csv`

### Install dependencies
```bash
pip install -r requirements.txt
```

### Train the model
```bash
python src/train.py
```

### Start the FastAPI backend
```bash
python -m uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```
API docs: `http://localhost:8000/docs`

### Start the Frontend
Since it's a native web app, you can just open `public/index.html` in your browser, or use a local server:
```bash
python -m http.server 8501 --directory public
```
UI: `http://localhost:8501`
